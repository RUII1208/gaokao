This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

贵州省2025年高考志愿填报助手 — a zero-dependency single-page web app that helps Guizhou gaokao students find universities matching their scores. Data comes from official 2025 admission PDFs, extracted via Python and baked into a JS file.

## Running

Open `index.html` directly in a browser — no server, build step, or dependencies needed. The app loads `universities_data.js` via a relative `<script src>`, so a `file://` URL works fine.

## Architecture

**Single-file app**: All HTML, CSS, and JS live in `index.html`. University data is in a separate `universities_data.js` loaded via `<script src>`.

- **CSS** (lines 7–544): Inline `<style>` with CSS custom properties. Includes print stylesheet and mobile responsive breakpoints at 640px.
- **HTML** (lines 546–651): Hero banner → subject selector chips → score/rank/region/tier inputs → search button → `#results` container → footer.
- **JS** (lines 653–941): Global functions, no modules. `search()` filters and classifies universities into rush/stable/safe categories based on score vs `avgScore`. `render()` generates all result HTML via template literals.

**Data file** (`universities_data.js`, ~18,500 lines, ~1MB): Contains `const universitiesPhysics` and `const universitiesHistory` arrays, plus `getUniversities()` which returns the correct array based on `selectedPrimary` (物理 or 历史). Generated by `generate_js.py` — do not edit manually.

## Key Data Flow

1. User selects primary subject (物理/历史) → determines which data array to use
2. User enters score (required) and optional rank, region, tier filters
3. `search()` calls `getUniversities()` → filters by region/tier/score (within 30 points below avgScore) → classifies as rush/stable/safe → sorts by category then score
4. Classification uses `avgScore` (not `minScore`): rush = score < avgScore-10, stable = ±10, safe = score > avgScore+10. Rank-based override applies when rank is provided.

## Regenerating Data

Source PDFs contain official admission data (not committed to git per `.gitignore`). To regenerate `universities_data.js`:

1. Place the 4 PDF files in the project root
2. Run `python generate_js.py` (requires `pdfplumber` and `PyMuPDF` pip packages)
3. The script reads `university_data_2025.json` (intermediate file) and writes `universities_data.js`

The `generate_js.py` script uses hardcoded `TIER_MAP` (name substring → tier) and `REGION_MAP` dictionaries to classify universities. Tier detection is approximate — it matches by university name substrings, not official codes. For example, `北京大学` falls through to `"普通本科"` because `"北京"` alone isn't in `TIER_MAP` (only `"北京航空"`, `"北京理工"`, etc. are). This is a known data quirk.

## University Object Shape

```js
{
  name: "清华大学", code: "1812", tier: "985", region: "其他",
  type: "公办", minScore: 676, avgScore: 679, minRank: 56,
  track: "物理",
  majors: [{ name: "理科试验班", score: 681, rank: 59 }, ...]  // max 15, sorted desc by score
}
```

Tier values: `"985"`, `"211"`, `"双一流"`, `"普通本科"`. Region values: Chinese province/city names or `"其他"`.
